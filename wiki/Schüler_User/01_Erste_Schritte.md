# Erste Schritte

Dieses Kapitel hilft dir beim Einstieg in Nova School.

## Anmeldung

Melde dich mit deinem Benutzernamen und Passwort an. In Demo-Umgebungen gibt es oft vorbereitete Testkonten.

Beispiel aus der Standardinstallation:

| Rolle | Benutzername |
| --- | --- |
| Lehrkraft | `teacher` |
| Schüler | `student` |

## Was du nach der Anmeldung siehst

Typische Bereiche der Oberfläche:

- Projektliste
- Dateibereich
- Editor
- Ausgabefeld
- Notebook-Zellen
- Dokumentation
- KI-Hilfe oder Mentor
- Chat
- Review- und Abgabebereich

## Dein persönlicher Profilordner

Jeder Benutzer bekommt einen eigenen Arbeitsbereich auf dem Server. Dort liegen:

- deine persönlichen Projekte
- deine Dateien
- deine Notebook-Zellen
- technische Metadaten für Ausführung und Kollaboration

## Persönliche Projekte und Gruppenprojekte

Es gibt zwei typische Besitzarten:

| Typ | Bedeutung |
| --- | --- |
| persönliches Projekt | nur dir zugeordnet |
| Gruppenprojekt | gemeinsames Projekt deiner Lerngruppe |

Ob du Gruppenprojekte nutzen kannst, hängt von deinen Rechten und deiner Gruppenzugehörigkeit ab.

## Ein Projekt anlegen

Typischer Ablauf:

1. Projektformular öffnen
2. Projektname eintragen
3. Vorlage auswählen
4. Besitzart auswählen
5. bei Gruppenprojekten die passende Gruppe wählen
6. Projekt anlegen

## Schnellstart-Vorlagen

Je nach Freigabe kannst du mit Vorlagen arbeiten, zum Beispiel:

- Python
- JavaScript
- C++
- Java
- Rust
- Frontend-Labor
- Distributed Playground

## Wenn etwas ausgegraut oder unsichtbar ist

Dann fehlt meistens ein Recht. Beispiele:

- kein Webzugriff
- kein KI-Zugriff
- keine Freigabe für bestimmte Sprache
- keine Gruppenberechtigung

In diesem Fall:

- nicht improvisieren
- Lehrkraft fragen, welche Freigabe für deine Aufgabe vorgesehen ist

## Gute Startreihenfolge

1. Projekt öffnen
2. Dateibaum prüfen
3. Hauptdatei öffnen
4. kleinen Testlauf machen
5. erst danach mit größerer Logik anfangen
